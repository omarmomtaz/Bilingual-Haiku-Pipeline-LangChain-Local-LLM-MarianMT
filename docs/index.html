01 — The Problem

Three failures.
Three engineering lessons.

Every fix made the system more robust. This is what separates a portfolio project from production engineering.

01
🧠
LLM Can't Enforce Structure
TinyLlama was asked to output 3 lines with a 5-7-5 syllable pattern. Every run returned prose, examples, and meta-commentary about the task — not the task itself.
Fix → Python enforces structure. LLM generates creative words. Each tool does what it was trained to do.
02
⚡
Broken Framework Abstraction
pipeline("translation") threw a KeyError. HuggingFace removed the translation task from the registry entirely in transformers v5.
Fix → Call MarianMTModel directly. One level below the abstraction. Always works.
03
🔤
Sentence Model, Paragraph Input
Feeding the full multi-stanza poem to MarianMT collapsed all stanza structure into one unbroken block of Japanese text.
Fix → Translate line-by-line. Reassemble stanzas in Python. Match input format to training data.
02 — Architecture

How it works.
Every component earns its place.

Step 01 · TinyLlama
Imagery Word Generation
TinyLlama-1.1B-Chat is invoked via LangChain LCEL to generate short vivid nature words per topic. Short word lists — not structure — are exactly what small LLMs do reliably. The output extends a curated imagery bank used in Step 2.
imagery_prompt = PromptTemplate.from_template("Give 3 short nature words...")
imagery_chain = imagery_prompt | llm | StrOutputParser()
extra_words = imagery_chain.invoke({"topic": topic})
Step 02 · Python Template Engine
5-7-5 Haiku Assembly
Pre-built haiku templates with {A}, {B}, {C} slots are filled with imagery words. Crucially, {B} IS the entire 7-syllable middle line — never embedded mid-sentence. This guarantees clean, grammatical English every run.
# {B} = the entire line 2. Never embedded mid-sentence.
TEMPLATES = [
  ("first light on {A}", "{B}", "cold {C} gleams"),
  ("the last {A} falls", "{B}", "{C} is gone"),
]
Step 03 · Helsinki-NLP MarianMT
Line-by-Line Translation
MarianMT is called directly (bypassing the broken pipeline abstraction) and translates each line individually. Stanzas are reassembled in Python after translation. This preserves the visual structure of the poem exactly.
# Direct model call — version-stable, full control
inputs = mt_tokenizer([line], return_tensors="pt", ...)
with torch.no_grad():
  out = mt_model.generate(**inputs)
result = mt_tokenizer.decode(out[0], skip_special_tokens=True)
